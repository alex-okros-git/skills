# Meta Landing Page — Smart Clone System

A two-step prompt system that lets you reverse-engineer the **conversion logic** of any high-performing landing page, then rebuild it as a structurally original page — without copying a single layout, section order, or visual element.

## Why This Exists

Copying a landing page is easy and useless. The layout isn't what makes it convert — the **psychological sequence** is. This system separates the two:

1. **Extract the logic** — what the page makes you *believe*, and in what order.
2. **Rebuild from logic alone** — with hard constraints that force structural originality.

The result is a page that converts for the same reasons as the source, but looks and flows completely differently.

## How It Works

```
┌─────────────┐      Principles      ┌─────────────┐      Landing      ┌──────────────┐
│ extract.md  │ ───── Brief ───────▶ │  build.md   │ ───── Page ─────▶ │  Output HTML │
│ (Step 1)    │                      │  (Step 2)   │                   │              │
└─────────────┘                      └──────┬──────┘                   └──────────────┘
                                            │
                                     ┌──────┴──────┐
                                     │  DESIGN.md  │
                                     │ (Optional)  │
                                     └─────────────┘
```

### Step 1 → `extract.md`

Feed an AI model the landing page you want to learn from (URL, copy-paste, or screenshot). The prompt forces the model to ignore all visual elements and answer six questions about the page's **persuasion architecture**:

- What does the visitor already believe?
- What gets established first, and why?
- Where and how are objections preempted?
- What trust mechanism is used (proof, authority, specificity, etc.)?
- When does the price appear relative to what's been established?
- What emotional state does the final push before the CTA produce?

The output is a numbered **Principles Brief** — a pure logic document with zero visual description.

### Step 2 → `build.md`

Paste the Principles Brief into this prompt. The model builds a complete landing page that applies the same conversion logic, but with **four hard structural constraints**:

| Constraint | Rule |
|---|---|
| **Hero type** | Must differ from the source (video → text, split-screen → stats, etc.) |
| **Section order** | Source order is off-limits — invent a new sequence |
| **Proof format** | If source used testimonials, use stats/case studies — and vice versa |
| **CTA position** | Place it somewhere the source did not |

Before writing code, the model must write a paragraph confirming it differs on all four.

### Optional → `DESIGN.md`

A [DESIGN.md spec](https://github.com/google-labs-code/design.md/) file containing your brand's design: colors, typography, spacing, shadows, border radii, and component definitions. When provided, the builder applies your visual identity to the output automatically.

The included `DESIGN.md` is pre-configured for **Brico Depot** (signature red `#E30612`, Inter typography, 8px grid, light/dark mode). Replace it with your own brand tokens or delete it to get unstyled output.

> **Note:** The included DESIGN.md was generated by adapting the output of the [design-system-when-you-have-no-designers-around](../design-system-when-you-have-no-designers-around/) skill.

## Files

| File | Purpose |
|---|---|
| `extract.md` | Step 1 prompt. Extracts conversion logic from any landing page |
| `build.md` | Step 2 prompt. Builds a structurally original page from the logic |
| `DESIGN.md` | Optional brand tokens. Visual identity applied to the output |

## Quick Start

1. Open your AI coding assistant (Claude, Gemini, ChatGPT, Cursor, etc.)
2. Paste the contents of **`extract.md`** and provide the landing page you want to study
3. Copy the numbered Principles Brief from the output
4. Paste the contents of **`build.md`** into a new conversation, inserting the brief where indicated
5. *(Optional)* Include your `DESIGN.md` contents for branded output
6. Get a structurally original, conversion-optimized landing page

## Adapting for Your Brand

To use this with your own brand instead of Brico Depot:

1. Replace the `DESIGN.md` file with your own design tokens (follow the [DESIGN.md spec](https://github.com/google-labs-code/design.md/blob/main/docs/spec.md))
2. Or generate one using the [design-system-when-you-have-no-designers-around](../design-system-when-you-have-no-designers-around/) skill
3. Uncomment the DESIGN.md section at the bottom of `build.md` and paste your tokens

Happy building.
